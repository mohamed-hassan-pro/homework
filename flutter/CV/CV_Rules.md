# 📋 Professional Resume Writing Rules
> Reference document for all future CV/Resume generation. Every resume MUST comply with these rules.

---

## 1. ⏱️ THE 6-SECOND RULE

Recruiters spend **6–9 seconds** on an initial scan. Everything on the resume must survive this filter.

**Implications:**
- The most important info (name, summary, strongest skill) must be in the **top third** of the page
- Bullets must be scannable — recruiter reads **first 3-4 words** of each bullet only
- Put the **keyword/metric FIRST** in each bullet, not buried at the end
- Maximum **1 page** for interns/fresh graduates, no exceptions
- Use **white space** generously — a crowded page gets skipped

---

## 2. 📐 SECTION ORDER (Student/Intern)

```
1. Header (Name + Contact)
2. Professional Summary (2-3 sentences MAX)
3. Education (students put this HIGH — it's your biggest asset)
4. Experience (reverse chronological)
5. Projects (only if directly relevant to the role)
6. Technical Skills (hard skills ONLY — categorized)
7. Additional (awards/competitions — only if space allows)
```

> **Rule:** If you have real work experience, it goes BEFORE projects.
> Education goes high because you're a student — once you have 2+ years of experience, move it to the bottom.

---

## 3. ✍️ THE X-Y-Z BULLET FORMULA (Google Standard)

Every bullet under Experience or Projects MUST follow:

```
"Accomplished [X] as measured by [Y], by doing [Z]."
```

- **X** = The accomplishment (what you did)
- **Y** = The measurable impact (number, %, time saved, result)
- **Z** = The technical method (how you did it)

**But keep it SHORT:** Target **1 line per bullet, max 2 lines.** Under 20 words is ideal.

### Examples:
- ❌ "Responsible for designing the database and handling backend integration and sync"
- ✅ "Designed the database schema (ERD), reducing data inconsistencies to zero post-launch"

---

## 4. 🚫 WORDS TO NEVER USE (Overused/Filler/Empty)

### Empty Trait Claims (claims without proof):
| ❌ Never Write | Why |
|---|---|
| Hard worker | Unprovable — show it through results instead |
| Dedicated / Passionate | Every applicant says this |
| Detail-oriented | Prove it with a metric (e.g., "100% data accuracy") |
| Self-starter | Show it through initiative stories |
| Fast learner | Show it through "promoted in 3 months" |
| Results-driven | The results themselves should prove this |
| Go-getter | Means nothing |
| Team player | Show collaboration in bullet points instead |

### Weak/Passive Verbs:
| ❌ Avoid | ✅ Replace With |
|---|---|
| Responsible for | Designed, Engineered, Built |
| Assisted / Helped | Collaborated, Contributed to, Co-developed |
| Handled | Managed, Orchestrated, Executed |
| Worked on | Developed, Implemented, Delivered |
| Participated in | Led, Drove, Coordinated |
| Supported | Enabled, Strengthened, Improved |
| Was involved in | Spearheaded, Initiated, Owned |
| Utilized / Leveraged | *(just name the tool directly)* |

### Filler Phrases to Delete:
| ❌ Delete | Why |
|---|---|
| "Proven track record" | Cliché — the bullets themselves are the proof |
| "Strong communication skills" | Unverifiable claim |
| "Excellent problem-solving abilities" | Show don't tell |
| "References available upon request" | Never include this — they'll ask |
| "Seeking a challenging position" | Outdated objective language |
| "Successfully" | Redundant — if you did it, it was successful |
| "Various" / "Multiple" | Be specific — say exactly how many |
| "Etc." | Never — either list it or don't |

---

## 5. 💪 STRONG ACTION VERBS (Use These)

### Technical / Building:
`Architected` · `Engineered` · `Designed` · `Developed` · `Built` · `Implemented` · `Integrated` · `Automated` · `Deployed` · `Optimized` · `Refactored`

### Problem-Solving:
`Resolved` · `Debugged` · `Diagnosed` · `Overhauled` · `Streamlined` · `Eliminated`

### Leadership / Strategy:
`Led` · `Spearheaded` · `Directed` · `Coordinated` · `Mentored` · `Initiated` · `Pioneered`

### Analysis / Quality:
`Analyzed` · `Validated` · `Assessed` · `Identified` · `Evaluated`

> **Rule:** Never repeat the same verb twice on the entire resume.

## 6. 🎯 TAILORING RULES (Per Job Application)

### Keyword Extraction Process:
1. Read the JD sections: "What You'll Do" + "Requirements" + "Nice to Have"
2. Extract **exact phrases** — ATS is literal ("system integration" ≠ "integrating systems")
3. Use the **same phrasing** as the JD in your bullets and skills
4. Place critical keywords in: Summary (2-3), Skills section, first 3 words of bullets

### Honesty Filter:
> **CRITICAL RULE:** Never emphasize a skill you can't defend in a 30-minute interview.
> If you're weak at X, mention it once in skills but don't build bullets around it.
> If you're strong at Y, make it the centerpiece of your bullets.

### What to Include vs. Exclude:

| ✅ Include | ❌ Exclude |
|---|---|
| Experience directly relevant to the role | Every job you've ever had |
| Projects that demonstrate JD keywords | Learning exercises / toy projects |
| Metrics and numbers | Vague claims about quality |
| Hard skills matching the JD | Soft skills as a standalone list |
| Awards directly relevant | Every certificate you own |

---

## 7. 🧩 SOFT SKILLS RULE

### ❌ NEVER list soft skills as a standalone bullet list
```
# BAD:
Soft Skills: Communication, Leadership, Teamwork, Problem Solving...
```

### ✅ PROVE soft skills through your bullet points
```
# GOOD — This PROVES leadership, communication, and teamwork:
"Led a 3-person team to deliver an E-commerce app in a 2-hour sprint, winning 3rd place"
```

### Where to demonstrate soft skills:
- **Summary:** Mention 1-2 naturally (e.g., "collaborative engineer")
- **Bullets:** The action + result proves the soft skill without naming it
- **Never** dedicate a section to listing soft skills — it wastes space and looks generic

---

## 8. 📏 FORMATTING RULES

### Layout:
- **Single-column layout** — multi-column confuses ATS parsers
- **Standard headings** — "Experience", "Education", "Skills" (not creative names)
- **Font:** Arial, Calibri, Roboto, or Aptos — 10-12pt body, 14-16pt headers
- **Margins:** 0.5 to 1 inch
- **File format:** `.docx` or clean text-based PDF

### Bullet Points:
- **1-2 lines max** per bullet — if it hits 3 lines, split or trim
- **Under 20 words** is the sweet spot
- **One idea per bullet** — don't combine two achievements
- **No full sentences** — start with action verb, no "I" or "my"
- **No periods at end** ← pick one style and be consistent
- **2-4 bullets per role** — more than 4 means you're not prioritizing

### Tense:
- **Current role:** Present tense ("Design...", "Build...")
- **Past roles:** Past tense ("Designed...", "Built...")

### Numbers:
- Always use **digits** not words: "3 services" not "three services"
- Put the **number first** when possible: "70% reduction" not "reduced by 70%"

---

## 9. 🚫 WHAT TO NEVER INCLUDE

- ❌ Photo / headshot
- ❌ Age, gender, marital status, religion
- ❌ "References available upon request"
- ❌ Salary information
- ❌ Full sentences with "I", "my", "me"
- ❌ Generic objective statements
- ❌ Skills you can't defend in an interview
- ❌ Every certificate you own (only relevant ones)
- ❌ Irrelevant work history
- ❌ Spelling & grammar errors (proofread 3x)

---

## 10. ✅ PRE-SUBMIT CHECKLIST

For every bullet point:
- [ ] Starts with a strong, unique action verb?
- [ ] Contains a metric or measurable result?
- [ ] Is 1-2 lines max (under 20 words)?
- [ ] Most important keyword is in the first 3-4 words?
- [ ] Follows X-Y-Z formula?
- [ ] No weak verbs (responsible for, assisted, handled)?
- [ ] No empty trait claims (hard worker, passionate)?

For the full resume:
- [ ] One page only?
- [ ] Single-column layout?
- [ ] Standard section headings?
- [ ] Every skill listed can be defended in interview?
- [ ] Keywords from JD are used with exact phrasing?
- [ ] No verb is repeated twice?
- [ ] Consistent formatting (tense, punctuation, spacing)?
- [ ] Soft skills are SHOWN in bullets, not listed?
